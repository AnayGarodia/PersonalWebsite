mcpaudit

2025 – Present · Python · Published on PyPI
Static analysis tool for MCP server security
↗ GitHub ↗ PyPI: mcp-vulnscan

The Model Context Protocol has become the standard way to give language models access to tools — databases, filesystems, APIs, shell commands. But MCP servers are code, and code has bugs. Security bugs. The same tool that lets Claude query a database can be turned into an injection vector if the server developer isn't careful.

mcpaudit is a static analysis tool that scans MCP server source code for vulnerabilities before they reach production. It uses Python's ast module to parse source files and perform data flow analysis — no execution required, no network calls. You point it at a directory and it tells you what's wrong.

What it catches

The scanner detects 12 vulnerability categories organized around three threat models: code injection (shell, SQL, template, XML, LDAP, code), data handling issues (path traversal, SSRF, unsafe deserialization), and LLM-specific risks (prompt injection, log injection). It also catches hardcoded secrets — API keys, passwords, tokens embedded in source code.

Severity is context-sensitive. A subprocess.run(cmd, shell=True) call in a utility function gets classified medium. The same call inside a function decorated with @mcp.tool() — where the argument comes from user input via the LLM — gets classified high. The scanner tracks import aliases (import subprocess as sp) and detects patterns through function scope boundaries.

How it works

The core analysis walks the Python AST looking for patterns that match known vulnerability signatures. Each checker implements a visitor that looks for specific node types — Call nodes for function calls, Assign nodes for secret patterns, JoinedStr for f-string injection. The tool context classifier identifies which functions are reachable from MCP tool handlers.

Findings include the vulnerability type, CWE identifier, file path and line number, and a human-readable description. Output formats include plain text, JSON, and SARIF — the last of which integrates directly with GitHub Code Scanning for automated CI/CD pipelines. A baseline mode lets you establish a security snapshot and only alert on new findings.

CI/CD integration

The SARIF output format uploads directly to GitHub's code scanning dashboard via the github/codeql-action/upload-sarif action. Baseline mode is designed for this: run once to establish a snapshot of existing findings, then gate PRs on new ones only. Developers can suppress specific findings with inline comments referencing CWE identifiers when a finding is a known false positive.

The tool has no external dependencies at analysis time — it's pure Python using only the standard library's ast module. This means it runs in any CI environment without network access.
