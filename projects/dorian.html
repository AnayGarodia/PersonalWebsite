Dorian

2025 – Present · React + Node.js · Visual AI Agent Builder
↗ GitHub

Most AI agent frameworks are code-first: you write Python, import a library, wire functions together. Dorian is canvas-first: you drag nodes onto a board, connect them with edges, and the agent emerges from the graph. The goal is to make agent design legible — something you can reason about visually before you reason about it in code.

The name is a reference to Dorian Gray's portrait: the visible surface of a thing that contains its hidden logic.

How it works

The frontend is a React application built around a node-edge canvas. Each node represents a component in the agent pipeline: a trigger (user input or scheduled event), an LLM call, a tool invocation, a memory read/write, a condition branch, or an output. Edges represent data flow. You connect them by dragging from output ports to input ports.

The backend is a Node.js server that receives the serialized graph from the frontend and executes it. Execution is a topological traversal of the DAG: nodes with no incoming edges run first, then their downstream nodes, until the graph is exhausted or a termination condition is reached. Each node type has a handler registered on the backend that defines how it runs.

Design decisions

The canvas uses a custom renderer rather than a library like React Flow. This was intentional: the rendering needs to be tightly coupled to the execution model, and third-party graph libraries come with opinions about what graphs are that don't always align with what agent workflows need. The custom renderer is about 800 lines of React + canvas code.

Service integrations are implemented as node types on the backend. Adding a new integration means registering a handler: a function that takes the node's configuration and input data and returns output data. Current integrations include OpenAI/Anthropic API calls, web search, code execution in a sandboxed environment, and a basic vector memory store. The integration surface is intentionally narrow — Dorian is not trying to be a universal connector, just a way to compose agents.

The project is ongoing. The next piece is a debugger: a way to step through graph execution, inspect intermediate values at each node, and understand why an agent produced a particular output.
